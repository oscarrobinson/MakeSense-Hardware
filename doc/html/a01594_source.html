00045 /** Function to read a single bit from an i2c address. Wrapper for the
00046  * i2c dev function...
00047  */
00048 void readBit(uint8_t slave_address, uint8_t reg_address, uint8_t bitNum, uint8_t *buf) {
00049    uint8_t set[] = {reg_address, 0};
00050    i2c_transmitinit( slave_address, 2, set );
00051    while(!i2c_transferred()); /* Wait for transfer */
00052      
00053    // Create buffer to store result into:
00054    int BUF_SIZE = 9;
00055    uint8_t buffer[BUF_SIZE];
00056      
00057    i2c_receiveinit( slave_address, 2, buffer ); //need to receive two bytes, otherwise we get ERROR I2C: Arbitration lost
00058    while(!i2c_transferred()); /* Wait for transfer */
00059    // Get queried bit number from buffer and return result:
00060    &buf = 0 | buffer[bitNum]; // not sure if this is valid... (maybe + ?)
00061    //&buf = 0b0000000 buffer[bitNum];
00062 }

00117 /** Get gyroscope output rate divider.
00118  * The sensor register output, FIFO output, DMP sampling, Motion detection, Zero
00119  * Motion detection, and Free Fall detection are all based on the Sample Rate.
00120  * The Sample Rate is generated by dividing the gyroscope output rate by
00121  * SMPLRT_DIV:
00122  *
00123  * Sample Rate = Gyroscope Output Rate / (1 + SMPLRT_DIV)
00124  *
00125  * where Gyroscope Output Rate = 8kHz when the DLPF is disabled (DLPF_CFG = 0 or
00126  * 7), and 1kHz when the DLPF is enabled (see Register 26).
00127  *
00128  * Note: The accelerometer output rate is 1kHz. This means that for a Sample
00129  * Rate greater than 1kHz, the same accelerometer sample may be output to the
00130  * FIFO, DMP, and sensor registers more than once.
00131  *
00132  * For a diagram of the gyroscope and accelerometer signal paths, see Section 8
00133  * of the MPU-6000/MPU-9150 Product Specification document.
00134  *
00135  * @return Current sample rate
00136  * @see MPU9150_RA_SMPLRT_DIV
00137  */

01822  * Each 16-bit accelerometer measurement has a full scale defined in ACCEL_FS
01823  * (Register 28). For each full scale setting, the accelerometers' sensitivity
01824  * per LSB in ACCEL_xOUT is shown in the table below:
01825  *
01826  * <pre>

01828  * --------+------------------+----------------

01957  * and 3 on the auxiliary I2C interface. Data read by Slave 4 is stored in
01958  * I2C_SLV4_DI (Register 53).

01961  * defined in Register 25. This access rate can be reduced by using the Slave
01962  * Delay Enable registers (Register 103).
01963  *
01964  * External sensor data registers, along with the gyroscope measurement
01965  * registers, accelerometer measurement registers, and temperature measurement
01966  * registers, are composed of two sets of registers: an internal register set
01967  * and a user-facing read register set.
01968  *
01969  * The data within the external sensors' internal register set is always updated
01970  * at the Sample Rate (or the reduced access rate) whenever the serial interface
01971  * is idle. This guarantees that a burst read of sensor registers will read
01972  * measurements from the same sampling instant. Note that if burst reads are not
01973  * used, the user is responsible for ensuring a set of single byte reads
01974  * correspond to a single sampling instant by checking the Data Ready interrupt.
01975  *
01976  * Data is placed in these external sensor data registers according to
01977  * I2C_SLV0_CTRL, I2C_SLV1_CTRL, I2C_SLV2_CTRL, and I2C_SLV3_CTRL (Registers 39,
01978  * 42, 45, and 48). When more than zero bytes are read (I2C_SLVx_LEN > 0) from
01979  * an enabled slave (I2C_SLVx_EN = 1), the slave is read at the Sample Rate (as
01980  * defined in Register 25) or delayed rate (if specified in Register 52 and
01981  * 103). During each Sample cycle, slave reads are performed in order of Slave

01988  * change the higher numbered slaves' associated registers. Furthermore, if
01989  * fewer total bytes are being read from the external sensors as a result of
01990  * such a change, then the data remaining in the registers which no longer have

02000  * information regarding the characteristics of Slave 4, please refer to

02008  * will be associated with Slave 1. If Slave 2 is enabled as well, registers
02009  * starting from EXT_SENS_DATA_06 will be allocated to Slave 2.
02010  *
02011  * If Slave 2 is disabled while Slave 3 is enabled in this same situation, then
02012  * registers starting from EXT_SENS_DATA_06 will be allocated to Slave 3
02013  * instead.
02014  *
02015  * REGISTER ALLOCATION FOR DYNAMIC DISABLE VS. NORMAL DISABLE:
02016  * If a slave is disabled at any time, the space initially allocated to the
02017  * slave in the EXT_SENS_DATA register, will remain associated with that slave.
02018  * This is to avoid dynamic adjustment of the register allocation.
02019  *
02020  * The allocation of the EXT_SENS_DATA registers is recomputed only when (1) all
02021  * slaves are disabled, or (2) the I2C_MST_RST bit is set (Register 106).
